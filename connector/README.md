# CCC CUCM Connector

On-prem edge stack that records CUCM Built-In-Bridge calls with FreeSWITCH and
pushes them to the cloud portal. Three containers, host networking:

- **freeswitch** — shared image (`ghcr.io/jmetdev/ccc-freeswitch-docker`) plus
  bind-mounted CUCM BIB dialplan, ACL, and hook scripts generated by `install.sh`
- **connector** — local shim FreeSWITCH hooks POST to; transcodes/peaks and
  uploads media to the portal v2 API with the tenant `ccck_` token
- **whisper** — faster-whisper sidecar (compose profile `transcription`); claims
  local transcribe jobs and posts transcripts back through the connector

No separate FreeSWITCH/BIB repo is required. Keep the generic FreeSWITCH image;
per-tenant CUCM + transcription config is rendered at install time.

## Install (customer host)

Create a connector in the portal (**Settings → Connectors → New connector**),
enter your CUCM node IPs, then run the one-liner it shows you:

```bash
curl -fsSL https://raw.githubusercontent.com/jmetdev/ccc-recording-portal/main/connector/install.sh \
  | sudo bash -s -- \
      --token   ccck_XXXXXXXX \
      --portal  https://recorddev.cloudcorecollab.com \
      --cucm-nodes 172.25.100.10,172.25.100.11,172.25.100.30
```

Optional flags:

| Flag | Default | Purpose |
|------|---------|---------|
| `--cucm-nodes` | _(required)_ | CUCM node IPs → ACL + BIB dialplan |
| `--whisper-model` | `base` | faster-whisper model name |
| `--no-transcribe` | off | Skip the whisper container / queue |

`--cucm-nodes` is required. The installer:

1. Installs Docker-CE and creates `/opt/ccc-connector/{recordings,freeswitch,src}`
2. Downloads this connector bundle
3. Writes `freeswitch/acl.conf.xml` and renders `freeswitch/dialplan/cucm_bib.xml`
   from `freeswitch/dialplan/cucm_bib.xml.template`
4. Vendors BIB hook scripts into `freeswitch/scripts/`
5. Writes `.env` with portal token, ingest/worker secrets, `TRANSCRIBE`, `WHISPER_MODEL`
6. Starts FreeSWITCH (ESL healthcheck + `SYS_NICE`), connector, and whisper
   (`--profile transcription` when transcription is enabled)

Point CUCM Built-In-Bridge recording at this host **SIP `:5070`**, destination
**`1034`**. The connector shows as **Active** once it heartbeats.

## Data flow

```
CUCM BIB --RTP--> FreeSWITCH --records WAV--> shared volume
   FreeSWITCH hooks --HTTP--> connector (:9000)
   connector: ffmpeg m4a + peaks --HTTPS(ccck_)--> portal /api/v2/ingest
   connector queues transcribe job --> whisper sidecar
   whisper --HTTP--> connector /api/workers/transcripts --HTTPS--> portal
```

The connector spools work in SQLite (`/opt/ccc-connector/recordings/.connector/spool.db`)
so hook POSTs return instantly and uploads retry through network outages.

## FreeSWITCH image

`freeswitch` pulls the shared `ccc-freeswitch-docker` image (Sofia/SIP runtime).
CUCM BIB assets are **not** baked into that image — `install.sh` bind-mounts:

| Host path | Container path |
|-----------|----------------|
| `$DATA_DIR/freeswitch/acl.conf.xml` | `/etc/freeswitch/autoload_configs/acl.conf.xml` |
| `$DATA_DIR/freeswitch/dialplan/cucm_bib.xml` | `/etc/freeswitch/dialplan/cucm_bib.xml` |
| `$DATA_DIR/freeswitch/scripts/` | `/usr/local/sbin/` |

Compose also overrides the image healthcheck to pass the ESL password (bare
`fs_cli` otherwise flips the container unhealthy) and adds `cap_add: [SYS_NICE]`
to silence SCHED_FIFO/nice warnings.

## Transcription

Enabled by default (`TRANSCRIBE=true`). The whisper service:

- Builds from `connector/whisper/`
- Shares the recordings volume read-only
- Authenticates to the connector with `WORKER_TOKEN` (defaults to `INGEST_TOKEN`)
- Loads `WHISPER_MODEL` (default `base`) on CPU/int8

Disable with `--no-transcribe` (or `TRANSCRIBE=false` in `.env` and restart
without the `transcription` profile).

## Local test without CUCM

With the stack running you can exercise the pipeline by dropping a WAV into
`/opt/ccc-connector/recordings/` and POSTing the hooks the way FreeSWITCH would:

```bash
T=$(grep INGEST_TOKEN /opt/ccc-connector/src/.env | cut -d= -f2)
curl -s localhost:9000/api/ingest/start   -H "X-Ingest-Token: $T" -H 'content-type: application/json' \
  -d '{"refci":"test-1","near_addr":"1000","far_addr":"2000"}'
curl -s localhost:9000/api/ingest/complete -H "X-Ingest-Token: $T" -H 'content-type: application/json' \
  -d '{"refci":"test-1","files":{"stereo":"test.wav"},"duration_s":3}'
```
